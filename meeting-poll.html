📋 Full setup guide

Part A — Google OAuth Client ID (for Calendar access)

  1. Go to console.cloud.google.com → create a new project.
  2. APIs & Services → Library → search "Google Calendar API" → Enable.
  3. APIs & Services → OAuth consent screen → External → fill app name → Save.
  4. Add your Google account as a Test user on that same screen.
  5. Credentials → Create Credentials → OAuth client ID → Web application.
  6. Under Authorized JavaScript origins, add your hosting URL (e.g. https://yourusername.github.io) and http://localhost:8080 for local testing.
  7. Copy the Client ID and paste it into Step 2 above.

Part B — Google Apps Script backend (for persistent votes)

  1. Go to script.google.com → click New project.
  2. Delete the placeholder code and paste the entire contents of apps-script.js (provided alongside this file).
  3. Click Deploy → New deployment → Web app.
  4. Set Execute as: Me. Set Who has access: Anyone. Click Deploy.
  5. Authorize the permissions it requests (it will ask to access Google Sheets — that's expected).
  6. Copy the Web app URL and paste it into Step 4 above, then click "Test connection."
  7. A Google Sheet named MeetingPollVotes will be created automatically in your Drive on first use.

Part C — Hosting the HTML file

  1. GitHub Pages (recommended): Upload meeting-poll.html to a public repo → Settings → Pages → Deploy from main branch. URL: https://yourusername.github.io/reponame/meeting-poll.html
  2. Local testing: In Terminal, cd to the file's folder, run python3 -m http.server 8080, visit http://localhost:8080/meeting-poll.html.
