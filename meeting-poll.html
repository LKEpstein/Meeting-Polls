📋 Setup guide: getting your Google OAuth Client ID
  1. Go to console.cloud.google.com and create a new project (or use an existing one).
  2. In the left menu, go to APIs & Services → Library. Search for Google Calendar API and enable it.
  3. Go to APIs & Services → OAuth consent screen. Set to External, fill in the app name (e.g. "My Meeting Poll"), save.
  4. Go to APIs & Services → Credentials → Create Credentials → OAuth client ID.
  5. Select Web application. Under Authorized JavaScript origins, add the exact URL where this file lives — e.g. https://yourdomain.com or http://localhost:8080.
  6. Click Create. Copy the Client ID and paste it into the field in Step 2 above.
  7. Back on the OAuth consent screen, add your own Google account as a Test user (under "Test users") so you can authenticate before publishing.

To host this file: Upload meeting-poll.html to any static web host — GitHub Pages (free), Netlify (free), or simply open it locally with python3 -m http.server 8080 and visit http://localhost:8080/meeting-poll.html. The file must be served over HTTP (not opened as file://) for Google OAuth to work.
